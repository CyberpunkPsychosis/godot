# 学习路线:从零到"一个人 + AI 做出游戏"

这份文档回答三个问题:**整个开发流程长什么样、每个软件在流程里干什么、每个软件具体学哪一小块就够用**。

## 总流程(先看懂这张图)

```
想法
 └→ ① 原型:AI 控制台 + 占位素材,先把"好不好玩"跑出来   ← 不需要任何美术软件
     └→ ② 素材:免费库里找(AI 帮你搜) / 自己做(Aseprite/Blender)
         └→ ③ 动画:2D 精灵表 / 3D 自带动画的模型 / Mixamo 套动作
             └→ ④ 组装:回到 Godot,一句话让 AI 接线(切帧、挂模型、连动画)
                 └→ ⑤ 试玩迭代:play → 读报错 → 改 → 再玩(AI 全程代劳)
                     └→ ⑥ 打包发布(Godot 自带导出)
```

**核心心法:①⑤ 永远在 Godot + AI 控制台里做;②③ 才需要外部软件;④ 又回到 AI。
先用占位素材把玩法做对,再替换成真素材——顺序反了会浪费大量时间。**

---

## 第 0 周:什么都不装,先跑通原型

打开 Godot AI 控制台,直接对 AI 说:

1. "新建一个平台跳跃场景,建个角色,搭个地面" → 占位方块角色就能跑跳
2. "运行看看,有报错帮我修" → 闭环调试
3. 玩法不好玩就改数值:"跳跃再高一点,移动再快一点"

这一步的目的:熟悉"用嘴做游戏"的节奏,并验证你的玩法点子。

## 2D 路线

### Aseprite(像素画 + 逐帧动画)—— 入门约 2~3 小时

**只学这些就够用:**
- 画布/调色板/图层/铅笔与油漆桶(30 分钟)
- 动画帧 + 洋葱皮(Onion Skin)(30 分钟)
- **导出精灵表**:`File → Export Sprite Sheet`,布局选 By Rows,记住导出的行列数(这是要告诉 AI 的关键信息)

**怎么学**:官方文档很短;B 站/YouTube 搜「Aseprite 入门」看一个 20 分钟视频即可开画。

### PixelLab 插件(AI 生成半成品)

装进 Aseprite 后的典型循环:
1. 文字描述生成角色(比如 "knight with sword, side view")
2. 不满意的部位框选 → inpaint 局部重生成
3. 用它的骨架动画功能从一张站立图生成走/跑帧
4. 逐帧手修崩坏的地方(通常每个动画修 1~2 帧)

### 交回 Godot(和 AI 的交接点)

把导出的 png 精灵表放进项目 `assets/` 文件夹,然后对 AI 说:

> "assets/hero.png 是精灵表,8 列 2 行,第 0-3 帧是 idle,第 8-11 帧是 run,帮我做成带动画的角色"

AI 会自动:切帧(create_sprite_frames)→ 建角色(build_character_2d)→ 接好 idle/run/jump 自动切换。

## 3D 路线

### 免费素材直接拿(不用学任何软件)

- 对 AI 说 "找个带动画的低模角色" → 它给你 Quaternius 链接 → 下载 zip → "导入这个 zip,用里面的角色建玩家"
- 模型/材质:"找个木桶模型""给墙贴砖材质" → Poly Haven/ambientCG 全自动下载

### Blender(改模/转格式)—— 最小学习集约 3~5 小时

**第一阶段只学这些(够覆盖 90% 需求):**
- 视角操作:中键旋转、滚轮缩放、Shift+中键平移(30 分钟,必须练熟)
- 导入:`File → Import → FBX/glTF`
- 选中/移动 G/旋转 R/缩放 S,删除多余部件 X(1 小时)
- **导出 glTF**:`File → Export → glTF 2.0`,格式选 `.glb`,勾上 +Y Up(默认即可)

**怎么学**:B 站搜「Blender 4 入门 甜甜圈」是经典教程,但你不需要学完——前 2~3 集(界面+基本操作)就够;建模/雕刻/渲染都可以以后再说。

### Mixamo(人形自动绑骨 + 动作库)—— 零学习成本

流程(全程网页点点点):
1. mixamo.com 登录(Adobe 免费账号)
2. Upload Character → 传你的模型(FBX/OBJ,没有骨骼也行)
3. 按提示在下巴/手腕/肘/膝/腹部放 5 个标记点 → 自动绑骨
4. 左侧搜动作(walk / run / jump / attack...)→ 预览满意就 Download
   - **下载设置:Format 选 FBX Binary,Skin 选 With Skin;循环动作勾 In Place**
5. 多个动作:重复第 4 步逐个下载(角色相同勾 Without Skin 可减小体积)

### 交回 Godot(和 AI 的交接点)

- glb 文件:直接丢进项目 `assets/`,说 "用 assets/knight.glb 建一个能跑的角色" → AI 自动识别 idle/run/jump 动画片段接好
- FBX 文件:Godot 首次导入会提示下载官方 FBX 导入器,点确认即可;或者在 Blender 里 FBX→glb 转一道(更稳)
- 多个 Mixamo 动作合并到一个角色:在 Blender 里用 glTF 合并,或先用单动作模型跑通(进阶话题,需要时问 AI)

**常见坑**:Mixamo 导出的模型经常放大 100 倍——进场景后如果巨大,直接跟 AI 说"把 Model 缩小到 0.01"。

## 音频(最省力的一环)

- 音效:对 AI 说"找打击音效" → Kenney 音效包链接 → 下载导入;播放逻辑让 AI 写("跳跃时播放这个音效")
- 音乐:前期不做;后期可用 CC0 音乐站或 AI 音乐工具,拖进项目即可

---

## 建议的学习节奏

| 时间 | 做什么 |
|---|---|
| 第 1 周 | 纯 AI 控制台做原型(占位素材),每天让 AI 帮你加一个玩法点 |
| 第 2 周 | 装 Aseprite(+PixelLab),画/生成你的主角,替换 2D 占位;或走 3D:Quaternius 素材 + Mixamo 流程跑一遍 |
| 第 3 周 | Blender 最小学习集,能改模转格式;素材管线全打通 |
| 之后 | 按需深入:UI 美化、音频、关卡设计——每一步都先问 AI"这个用什么方案" |

**万能原则:卡住了就把问题原样发给 AI 控制台或 Claude Code——包括报错截图、"我想要 X 但不知道怎么做"。工具链的一切接线问题都是 AI 的活,你只负责创意和审美。**
